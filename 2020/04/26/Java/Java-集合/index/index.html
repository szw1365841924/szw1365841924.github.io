1.ArrayList详解

首先从他的成员属性入手。上图！

从上面的内容我们可以知道：

  1. ArrayList是基于Object数组构建的。
  2. 如果未指定容量大小，则默认容量是10。
  3. ArrayList的最大容量为Integer.MAX_VALUE - 8，如果超出这个大小会出现OOM(内存溢出)异常。

接下来我们先看ArrayList的构造方法！

1.无参构造方法

注意！这里所说的区分EMPTY_ELEMENTDATA和DEFAULT_EMPTY_ELEMENTDATA主要是指：如果使用指定初始容量的构造函数的话，传入的值为0则应该初始化一个长度为0的空数组。这时就要使用EMPTY_ELEMENTDATA。如果使用无参构造方法，虽然赋值的也是一个空数组，但是！在第一次添加元素的时候会判断下当前数组是否是DEFAULT_EMPTY_ELEMENTDATA数组，如果是那么先初始化成容量大小为10的数组。

看明白了吗？

总结下就是，如果指定初始数组容量为0，那么使用的是EMPTY_ELEMENTDATA。但是未指定的话，使用DAFAULT_EMPTY_ELEMENTDATA但是默认的初始化大小是10而非0。

add方法给了两个

前者是添加到ArrayList的末尾，后者是添加到指定位置。

我们先来看前者add(E e)：

public boolean add(E e) {
    //1. 先进行容量确认，查看size + 1后是否需要扩容
    ensureCapacityInternal(size + 1);  // Increments modCount!!
    //2. 将元素添加到数组
    elementData[size++] = e;
    return true;
}

这里调用了一个方法ensureCapacityInternal()，我们继续跟进：

private void ensureCapacityInternal(int minCapacity) {
	ensureExplicitCapacity(calculateCapacity(elementData, minCapacity));
}

这里又调用了ensureExplicitCapacity()，在方法参数中调用了calculateCapacity()。我们先进入calculateCapacity()中看一下。

private static int calculateCapacity(Object[] elementData, int minCapacity) {
    //1. 在这里先确认elementdata是否指向的是DEFAULTCAPACITY_EMPTY_ELEMENTDATA这个空数组
    //	 如果是，比较下minCapacity(其实就是size + 1后的大小)和DEFAULT_CAPACITY(10)谁大返回谁
    if (elementData == DEFAULTCAPACITY_EMPTY_ELEMENTDATA) {
    	return Math.max(DEFAULT_CAPACITY, minCapacity);
    }
    //	 如果不等于DEFAULTCAPACITY_EMPTY_ELEMENTDATA直接返回minCapacity
    return minCapacity;
}

在这里我们可以看到之前提到的，如果使用默认构造方法，虽然先赋值了一个空数组，但是在这里会扩容成大小为10的数组。

calculateCapacity()说完了，就该进入到ensureExplicitCapacity()了。

private void ensureExplicitCapacity(int minCapacity) {
    // 这个modCount不用管，我进入看了以下，认为就是记录ArrayList集合操作的次数
    modCount++;

    // 这里如果minCapacity(size + 1)大于elementdata数组的长度那么就进行扩容
    if (minCapacity - elementData.length > 0)
    	grow(minCapacity);
}

哎，又来一个grow()，其实如果当前的minCapacity(size + 1)没有超出数组的范围，这里就可以直接返回了。但是超出了就需要继续进入到grow()中。

private void grow(int minCapacity) {
    // 将数组的长度赋值给oldCapacity
    int oldCapacity = elementData.length;
    // newCapacity = 1.5倍的oldCapacity，这里用到 >> 移位运算符。(想到了大二被数电支配的恐惧...)相当于除2，也就是 oldCapacity >> 1 == oldCapacity / 2
    int newCapacity = oldCapacity + (oldCapacity >> 1);
    // 如果扩容后的数组长度仍然小于minCapacity(size + 1)，那就直接把minCapacity(size + 1)赋值给newCapacity，保证扩容后数组长度 >= minCapacity(size + 1)
    if (newCapacity - minCapacity < 0)
    	newCapacity = minCapacity;
    // 这里检查扩容后的数组大小是否大于ArrayList允许的最大值。如果超出了MAX_ARRAY_SIZE，那就要检查是否溢出。这里我在下面直接贴出hugeCapacity()了，简单看看。
    if (newCapacity - MAX_ARRAY_SIZE > 0)
    	newCapacity = hugeCapacity(minCapacity);
    // 将原先的数组复制到扩容后的新数组，再将引用传给elementdata。
    elementData = Arrays.copyOf(elementData, newCapacity);
}

private static int hugeCapacity(int minCapacity) {
    if (minCapacity < 0) // overflow
        throw new OutOfMemoryError();
    return (minCapacity > MAX_ARRAY_SIZE) ?
        Integer.MAX_VALUE :
    MAX_ARRAY_SIZE;
}

到这里add()就算彻底完成了……有一点点绕。我给大家画个图感受下。

remove()

// remove()有两个，一个是移除指定位置的元素，一个是移除指定元素。
//1. 移除指定位置元素，这个没什么后说的，先确定下标是否越界。没越界则移除指定位置元素。并将index后面的元素向前移动一个位置
public E remove(int index) {
    rangeCheck(index);

    modCount++;
    E oldValue = elementData(index);

    int numMoved = size - index - 1;
    if (numMoved > 0)
        System.arraycopy(elementData, index+1, elementData, index,
                         numMoved);
    elementData[--size] = null; // clear to let GC do its work

    return oldValue;
}
//2. 移除指定元素，这里是先遍历寻找是否有指定元素，如果有调用fastRemove()
 public boolean remove(Object o) {
     if (o == null) {
         for (int index = 0; index < size; index++)
             if (elementData[index] == null) {
                 fastRemove(index);
                 return true;
             }
     } else {
         for (int index = 0; index < size; index++)
             if (o.equals(elementData[index])) {
                 fastRemove(index);
                 return true;
             }
     }
     return false;
}
// fastRemove()，用来移除元素，并将index后面的元素向前移动一个位置
private void fastRemove(int index) {
    modCount++;
    int numMoved = size - index - 1;
    if (numMoved > 0)
        System.arraycopy(elementData, index+1, elementData, index,
                         numMoved);
    elementData[--size] = null; // clear to let GC do its work
}

set()

// set()方法用来更新指定位置的数据
public E set(int index, E element) {
    // 判断下标是否越界
    rangeCheck(index);
	// 没有越界，更新指定位置元素，并返回旧元素值
    E oldValue = elementData(index);
    elementData[index] = element;
    return oldValue;
}

ArrayList总结

  • ArrayList是基于动态数组(数组+扩容机制)实现的，在增删的时候会涉及数组的赋值，会影响性能
  • ArrayList默认容量为10，扩容时先扩容1.5倍。
  • 根据remove(Object o)我们发现，ArrayList可以存放null值。
  • ArrayList内部实现时，并没有涉及到线程安全的问题，当多线程操作同一个ArrayList时会出现线程安全问题。如果想保证线程安全请使用CopyOnWriteArrayList

首先，还是从成员变量入手吧！

这里我们看一下Node这个静态内部类

// 也很简单，Node中包含一个存储内容的E，一个前驱指针prev，一个后继指针next
private static class Node<E> {
    E item;
    Node<E> next;
    Node<E> prev;

    Node(Node<E> prev, E element, Node<E> next) {
        this.item = element;
        this.next = next;
        this.prev = prev;
    }
}

/**
 * Links e as last element.
 * 官方文档也说的很清楚，把节点e作为最后一个元素连接在链表上。
 */
void linkLast(E e) {
    // 这里说实话也没什么好解释的，就是简单的双向链表操作。而且还是连接在尾部，操作更简单了。
    // l指向尾节点
    final Node<E> l = last;
    // 创建新新节点，新节点的前驱是l，后继为null
    final Node<E> newNode = new Node<>(l, e, null);
    // 把链表的尾指针指向新节点
    last = newNode;
    // 如果l等于null的话，就证明目前链表是空的，newNode是第一个节点
    if (l == null)
        // 让头指针指向新节点
        first = newNode;
    else
        // l不等于null的话，就证明目前链表有元素，直接让l的后继变成newNode就完成了连接
        l.next = newNode;
    // 元素个数加1
    size++;
    modCount++;
}

remove()

其实LinkedList作为双向链表来说有remove()，作为队列来说有poll()和remove()。

remove()和poll()区别在于，如果要移除的元素不存在，那么remove()会抛出异常，poll()会返回null。

这里我是作为双向链表来说的~所以说一下remove()吧，就remove()就有好多种。如removeFirst()，removeLast()。而且根据传入参数的不同remove()还有移除指定位置上的元素remove(int index)，移除指定元素remove(Object o)，这里我就说一下remove(Object o)

public boolean remove(Object o) {
    // 如果o为null，
    if (o == null) {
        // for循环中寻找null元素
        for (Node<E> x = first; x != null; x = x.next) {
            if (x.item == null) {
                // 拆解元素
                unlink(x);
                return true;
            }
        }
    } else {
        // 如果o不为null，for循环中寻找元素o
        for (Node<E> x = first; x != null; x = x.next) {
            if (o.equals(x.item)) {
                // 拆解元素
                unlink(x);
                return true;
            }
        }
    }
    return false;
}

// 这里给一下unlink()
/**
 * Unlinks non-null node x.
 * 解绑一个非空元素，ps：官方文档写的真专业。希望自己也可以成为这样的程序员
 */
E unlink(Node<E> x) {
    // 声明 x != null;因为找到了指定元素才会进入这个方法...所以x != null，我是这样理解的，如果有不一样的理解请务必联系我并指出！感谢感谢！
    final E element = x.item;
    // 拿到x的后继节点
    final Node<E> next = x.next;
    // 拿到x的前驱节点
    final Node<E> prev = x.prev;
	
    // 如果前驱节点为null，那就说明x为头节点，头指针first应该修改为x的后继节点
    if (prev == null) {
        first = next;
    } else {
        // 如果前驱不为null，那就将next作为前驱节点的后继节点
        prev.next = next;
        // 再将x的前驱节点变成null，断开连接
        x.prev = null;
    }
	
    // 如果后继节点为null，那就说明x为尾节点，尾指针last应该修改为x的前驱节点
    if (next == null) {
        last = prev;
    } else {
        // 如果后继不为null，那就将prev作为后继节点的前驱节点
        next.prev = prev;
        // 再将x的后继节点变成bull，断开连接
        x.next = null;
    }
	
    // 将x的内容变为null，便于JVM进行回收
    x.item = null;
    size--;
    modCount++;
    return element;
}

// 我们看下node()
/**
 * Returns the (non-null) Node at the specified element index.
 * 返回一个特殊位置的非空Node节点
 */
Node<E> node(int index) {
    // 这里有点意思，如果下标index小于size的一半(>> 相当于除于2)，就从头找
    if (index < (size >> 1)) {
        Node<E> x = first;
        for (int i = 0; i < index; i++)
            x = x.next;
        return x;
    // 大于size的一半就从尾找，充分利用了双向链表的特性！
    } else {
        Node<E> x = last;
        for (int i = size - 1; i > index; i--)
            x = x.prev;
        return x;
    }
}
