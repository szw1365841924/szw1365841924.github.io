1.无参构造方法

注意！这里所说的区分EMPTY_ELEMENTDATA和DEFAULT_EMPTY_ELEMENTDATA主要是指：如果使用指定初始容量的构造函数的话，传入的值为0则应该初始化一个长度为0的空数组。这时就要使用EMPTY_ELEMENTDATA。如果使用无参构造方法，虽然赋值的也是一个空数组，但是！在第一次添加元素的时候会判断下当前数组是否是DEFAULT_EMPTY_ELEMENTDATA数组，如果是那么先初始化成容量大小为10的数组。

看明白了吗？

总结下就是，如果指定初始数组容量为0，那么使用的是EMPTY_ELEMENTDATA。但是未指定的话，使用DAFAULT_EMPTY_ELEMENTDATA但是默认的初始化大小是10而非0。

这里又调用了ensureExplicitCapacity()，在方法参数中调用了calculateCapacity()。我们先进入calculateCapacity()中看一下。

private static int calculateCapacity(Object[] elementData, int minCapacity) {
    //1. 在这里先确认elementdata是否指向的是DEFAULTCAPACITY_EMPTY_ELEMENTDATA这个空数组
    //	 如果是，比较下minCapacity(其实就是size + 1后的大小)和DEFAULT_CAPACITY(10)谁大返回谁
    if (elementData == DEFAULTCAPACITY_EMPTY_ELEMENTDATA) {
    	return Math.max(DEFAULT_CAPACITY, minCapacity);
    }
    //	 如果不等于DEFAULTCAPACITY_EMPTY_ELEMENTDATA直接返回minCapacity
    return minCapacity;
}

在这里我们可以看到之前提到的，如果使用默认构造方法，虽然先赋值了一个空数组，但是在这里会扩容成大小为10的数组。

calculateCapacity()说完了，就该进入到ensureExplicitCapacity()了。

private void ensureExplicitCapacity(int minCapacity) {
    // 这个modCount不用管，我进入看了以下，认为就是记录ArrayList集合操作的次数
    modCount++;

    // 这里如果minCapacity(size + 1)大于elementdata数组的长度那么就进行扩容
    if (minCapacity - elementData.length > 0)
    	grow(minCapacity);
}

哎，又来一个grow()，其实如果当前的minCapacity(size + 1)没有超出数组的范围，这里就可以直接返回了。但是超出了就需要继续进入到grow()中。

private void grow(int minCapacity) {
    // 将数组的长度赋值给oldCapacity
    int oldCapacity = elementData.length;
    // newCapacity = 1.5倍的oldCapacity，这里用到 >> 移位运算符。(想到了大二被数电支配的恐惧...)相当于除2，也就是 oldCapacity >> 1 == oldCapacity / 2
    int newCapacity = oldCapacity + (oldCapacity >> 1);
    // 如果扩容后的数组长度仍然小于minCapacity(size + 1)，那就直接把minCapacity(size + 1)赋值给newCapacity，保证扩容后数组长度 >= minCapacity(size + 1)
    if (newCapacity - minCapacity < 0)
    	newCapacity = minCapacity;
    // 这里检查扩容后的数组大小是否大于ArrayList允许的最大值。如果超出了MAX_ARRAY_SIZE，那就要检查是否溢出。这里我在下面直接贴出hugeCapacity()了，简单看看。
    if (newCapacity - MAX_ARRAY_SIZE > 0)
    	newCapacity = hugeCapacity(minCapacity);
    // 将原先的数组复制到扩容后的新数组，再将引用传给elementdata。
    elementData = Arrays.copyOf(elementData, newCapacity);
}

private static int hugeCapacity(int minCapacity) {
    if (minCapacity < 0) // overflow
        throw new OutOfMemoryError();
    return (minCapacity > MAX_ARRAY_SIZE) ?
        Integer.MAX_VALUE :
    MAX_ARRAY_SIZE;
}

// remove()有两个，一个是移除指定位置的元素，一个是移除指定元素。
//1. 移除指定位置元素，这个没什么后说的，先确定下标是否越界。没越界则移除指定位置元素。并将index后面的元素向前移动一个位置
public E remove(int index) {
    rangeCheck(index);

    modCount++;
    E oldValue = elementData(index);

    int numMoved = size - index - 1;
    if (numMoved > 0)
        System.arraycopy(elementData, index+1, elementData, index,
                         numMoved);
    elementData[--size] = null; // clear to let GC do its work

    return oldValue;
}
//2. 移除指定元素，这里是先遍历寻找是否有指定元素，如果有调用fastRemove()
 public boolean remove(Object o) {
     if (o == null) {
         for (int index = 0; index < size; index++)
             if (elementData[index] == null) {
                 fastRemove(index);
                 return true;
             }
     } else {
         for (int index = 0; index < size; index++)
             if (o.equals(elementData[index])) {
                 fastRemove(index);
                 return true;
             }
     }
     return false;
}
// fastRemove()，用来移除元素，并将index后面的元素向前移动一个位置
private void fastRemove(int index) {
    modCount++;
    int numMoved = size - index - 1;
    if (numMoved > 0)
        System.arraycopy(elementData, index+1, elementData, index,
                         numMoved);
    elementData[--size] = null; // clear to let GC do its work
}

set()

// set()方法用来更新指定位置的数据
public E set(int index, E element) {
    // 判断下标是否越界
    rangeCheck(index);
	// 没有越界，更新指定位置元素，并返回旧元素值
    E oldValue = elementData(index);
    elementData[index] = element;
    return oldValue;
}

ArrayList总结

  • ArrayList是基于动态数组(数组+扩容机制)实现的，在增删的时候会涉及数组的赋值，会影响性能
  • ArrayList默认容量为10，扩容时先扩容1.5倍。
  • 根据remove(Object o)我们发现，ArrayList可以存放null值。
  • ArrayList内部实现时，并没有涉及到线程安全的问题，当多线程操作同一个ArrayList时会出现线程安全问题。如果想保证线程安全请使用CopyOnWriteArrayList

/**
 * Links e as last element.
 * 官方文档也说的很清楚，把节点e作为最后一个元素连接在链表上。
 */
void linkLast(E e) {
    // 这里说实话也没什么好解释的，就是简单的双向链表操作。而且还是连接在尾部，操作更简单了。
    // l指向尾节点
    final Node<E> l = last;
    // 创建新新节点，新节点的前驱是l，后继为null
    final Node<E> newNode = new Node<>(l, e, null);
    // 把链表的尾指针指向新节点
    last = newNode;
    // 如果l等于null的话，就证明目前链表是空的，newNode是第一个节点
    if (l == null)
        // 让头指针指向新节点
        first = newNode;
    else
        // l不等于null的话，就证明目前链表有元素，直接让l的后继变成newNode就完成了连接
        l.next = newNode;
    // 元素个数加1
    size++;
    modCount++;
}

其实LinkedList作为双向链表来说有remove()，作为队列来说有poll()和remove()。

public boolean remove(Object o) {
    // 如果o为null，
    if (o == null) {
        // for循环中寻找null元素
        for (Node<E> x = first; x != null; x = x.next) {
            if (x.item == null) {
                // 拆解元素
                unlink(x);
                return true;
            }
        }
    } else {
        // 如果o不为null，for循环中寻找元素o
        for (Node<E> x = first; x != null; x = x.next) {
            if (o.equals(x.item)) {
                // 拆解元素
                unlink(x);
                return true;
            }
        }
    }
    return false;
}

// 这里给一下unlink()
/**
 * Unlinks non-null node x.
 * 解绑一个非空元素，ps：官方文档写的真专业。希望自己也可以成为这样的程序员
 */
E unlink(Node<E> x) {
    // 声明 x != null;因为找到了指定元素才会进入这个方法...所以x != null，我是这样理解的，如果有不一样的理解请务必联系我并指出！感谢感谢！
    final E element = x.item;
    // 拿到x的后继节点
    final Node<E> next = x.next;
    // 拿到x的前驱节点
    final Node<E> prev = x.prev;
	
    // 如果前驱节点为null，那就说明x为头节点，头指针first应该修改为x的后继节点
    if (prev == null) {
        first = next;
    } else {
        // 如果前驱不为null，那就将next作为前驱节点的后继节点
        prev.next = next;
        // 再将x的前驱节点变成null，断开连接
        x.prev = null;
    }
	
    // 如果后继节点为null，那就说明x为尾节点，尾指针last应该修改为x的前驱节点
    if (next == null) {
        last = prev;
    } else {
        // 如果后继不为null，那就将prev作为后继节点的前驱节点
        next.prev = prev;
        // 再将x的后继节点变成bull，断开连接
        x.next = null;
    }
	
    // 将x的内容变为null，便于JVM进行回收
    x.item = null;
    size--;
    modCount++;
    return element;
}

/**
 * Returns the (non-null) Node at the specified element index.
 * 返回一个特殊位置的非空Node节点
 */
Node<E> node(int index) {
    // 这里有点意思，如果下标index小于size的一半(>> 相当于除于2)，就从头找
    if (index < (size >> 1)) {
        Node<E> x = first;
        for (int i = 0; i < index; i++)
            x = x.next;
        return x;
    // 大于size的一半就从尾找，充分利用了双向链表的特性！
    } else {
        Node<E> x = last;
        for (int i = size - 1; i > index; i--)
            x = x.prev;
        return x;
    }
}

static class Node<K,V> implements Map.Entry<K,V> {

    final int hash;
    // 你存放的key，可以看到key是不可变的
    final K key;
    // 你存放的value

        this.key = key;
        this.value = value;
        this.next = next;

        return key;
    }
    public final V getValue() { 
        return value;
    }
    public final String toString() {
        return key + "=" + value;
    }
	// 这里可以看到hash = key的hashcode ^ value的hashcode
    public final int hashCode() {
        return Objects.hashCode(key) ^ Objects.hashCode(value);
    }

    public final V setValue(V newValue) {
        V oldValue = value;
        value = newValue;
        return oldValue;
    }

    public final boolean equals(Object o) {
        if (o == this)
            return true;
        if (o instanceof Map.Entry) {
            Map.Entry<?,?> e = (Map.Entry<?,?>)o;
            if (Objects.equals(key, e.getKey()) &&
                Objects.equals(value, e.getValue()))
                return true;
        }
        return false;
    }
}

public HashMap(int initialCapacity, float loadFactor) {
    // 如果初始化容量小于0，抛出异常
    if (initialCapacity < 0)
        throw new IllegalArgumentException("Illegal initial capacity: " +
                                           initialCapacity);
    // 如果初始化容量大于HashMap的最大值(2的30次方)，直接赋值成最大值
    if (initialCapacity > MAXIMUM_CAPACITY)
        initialCapacity = MAXIMUM_CAPACITY;
    // 如果装载因子<=0或者装载因子不是数字，抛出异常
    if (loadFactor <= 0 || Float.isNaN(loadFactor))
        throw new IllegalArgumentException("Illegal load factor: " +
                                           loadFactor);
    this.loadFactor = loadFactor;
    // 这里调用了tableSizeFor()，下面我们看一下
    this.threshold = tableSizeFor(initialCapacity);
}

// 返回一个大于initialCapacity的并且最接近initialCapacity的2的整数次幂
// PS: 至于为什么要返回这个值，我也没有弄明白，按道理不应该是(initialCapacity * loadFactor)这个值吗？如果有了解的大佬还请指点以下啊！！！感谢！
static final int tableSizeFor(int cap) {
    int n = cap - 1;
    n |= n >>> 1;
    n |= n >>> 2;
    n |= n >>> 4;
    n |= n >>> 8;
    n |= n >>> 16;
    return (n < 0) ? 1 : (n >= MAXIMUM_CAPACITY) ? MAXIMUM_CAPACITY : n + 1;
}

final V putVal(int hash, K key, V value, boolean onlyIfAbsent,
                   boolean evict) {
    Node<K,V>[] tab; Node<K,V> p; int n, i;
    // 1. 如果tab为空或者长度为0，则先进行一次扩容
    if ((tab = table) == null || (n = tab.length) == 0)
        n = (tab = resize()).length;

    // (1) 当前是链表并且已经存在了即将插入的key，那就直接覆盖旧值，并返回旧值，而且因为是覆盖也不会发生扩容操作

    // 我们正式进入else中
    else {
        Node<K,V> e; K k;
        // 比较桶中(注意是桶，还没有进入桶中对应的链表或者树)第一个元素是否和即将插入的元素相同，相同则直接覆盖
        if (p.hash == hash &&
            ((k = p.key) == key || (key != null && key.equals(k))))
            e = p;
        // 这里走的就是(3)了，判断是否为树，如果是，则直接插入
        else if (p instanceof TreeNode)
            e = ((TreeNode<K,V>)p).putTreeVal(this, tab, hash, key, value);
        // 这里就是走的(1)和(2)
        else {
            // 遍历链表
            for (int binCount = 0; ; ++binCount) {
                // 判断是否到达尾部，如果到达尾部则证明整个链表中没有相同的key，那么就在尾部插入，并且跳出循环
                if ((e = p.next) == null) {
                    p.next = newNode(hash, key, value, null);
                    // 判断是否到达了阈值(8)，到达则尝试转化为红黑树，这里的treeifyBin()我有个地方要说下，一会大家看下面的源码
                    if (binCount >= TREEIFY_THRESHOLD - 1) // -1 for 1st
                        treeifyBin(tab, hash);
                    break;
                }
                // 在遍历时，判断当前节点的key是否和即将插入的的key相同，相同则覆盖，并且跳出循环
                if (e.hash == hash &&
                    ((k = e.key) == key || (key != null && key.equals(k))))
                    break;
                p = e;
            }
        }
        // 在桶中找到了该插入的位置，插入元素
        if (e != null) { // existing mapping for key
            V oldValue = e.value;
            if (!onlyIfAbsent || oldValue == null)
                e.value = value;
            afterNodeAccess(e);
            return oldValue;
        }
    }
    ++modCount;
    // 判断目前的元素个数是否超过了阈值(如：默认的最小是 16 * 0.75 = 12)，超过则扩容
    if (++size > threshold)
        resize();
    afterNodeInsertion(evict);
    return null;
}
/** 我为什么要说这里呢，因为我在学习的时候有一个点困扰了我，就是TREEIFY_THRESHOLD(默认8)和MIN_TREEIFY_CAPACITY(默认64)。
网上有的博客说当链表元素个数大于TREEIFY_THRESHOLD时，链表就会转化成红黑树。但是当我看见MIN_TREEIFY_CAPACITY我又有点迷惑了。
经过多方查询，其实应该是如下这样: 
我们认为当链表元素个数大于TREEIFY_THRESHOLD(默认8)时，进入到treeifyBin()，尝试转化成红黑树，这也和我们上面看到的源码相同。
进入到treeifyBin()后我们发现，有一句if判断当前的哈希表(就是table数组)的长度如果小于MIN_TREEIFY_CAPACITY，那么链表是不会变成红黑树的，优先进行扩容操作。
只有同时满足(链表长度 > REEIFY_THRESHOLD && 哈希表(table数组)的长度 > MIN_TREEIFY_CAPACITY)时才会将链表变成红黑树
以上就是我学习的个人理解，如果有不对的地方还请大家指出！！

final void treeifyBin(Node<K,V>[] tab, int hash) {
    int n, index; Node<K,V> e;
    // 这里就解开了我的困惑
    if (tab == null || (n = tab.length) < MIN_TREEIFY_CAPACITY)
        resize();
    else if ((e = tab[index = (n - 1) & hash]) != null) {
        TreeNode<K,V> hd = null, tl = null;
        do {
            TreeNode<K,V> p = replacementTreeNode(e, null);
            if (tl == null)
                hd = p;
            else {
                p.prev = tl;
                tl.next = p;
            }
            tl = p;
        } while ((e = e.next) != null);
        if ((tab[index] = hd) != null)
            hd.treeify(tab);
    }
}

这里还有一个很重要的地方，就是resize()这个扩容方法，我们来看一看~

// 有一说一，有点长，我还是建议仔细看看开头我给出的那篇博客，讲的很清楚~，这里我就标注一些我认为比较重要的部分
final Node<K,V>[] resize() {
    // oldTab指向table数组
    Node<K,V>[] oldTab = table; 
    int oldCap = (oldTab == null) ? 0 : oldTab.length;
    int oldThr = threshold;
    int newCap, newThr = 0;
    // 如果oldCap不为空的话，就是table数组不为空
    if (oldCap > 0) { 
        // 如果大于最大容量了，就调整阀值为整数的最大值
        if (oldCap >= MAXIMUM_CAPACITY) { 
            threshold = Integer.MAX_VALUE;
            // 直接返回
            return oldTab;
        }
        // 如果当前table数组长度变为2倍后，仍然小于HashMap的容量的最大值，并且oldcap(table数组的长度,此时还未改变所以用的oldcap而不是newcap)>=默认的阈值(默认16)，则把阈值扩大为2倍
        else if ((newCap = oldCap << 1) < MAXIMUM_CAPACITY &&
                 oldCap >= DEFAULT_INITIAL_CAPACITY)
            // 新的阈值是原先阈值的2倍
            newThr = oldThr << 1; // double threshold
    }
    else if (oldThr > 0) // initial capacity was placed in threshold
        newCap = oldThr;
    else {               // zero initial threshold signifies using defaults
        newCap = DEFAULT_INITIAL_CAPACITY;
        newThr = (int)(DEFAULT_LOAD_FACTOR * DEFAULT_INITIAL_CAPACITY);
    }
    if (newThr == 0) {
        float ft = (float)newCap * loadFactor;
        newThr = (newCap < MAXIMUM_CAPACITY && ft < (float)MAXIMUM_CAPACITY ?
                  (int)ft : Integer.MAX_VALUE);
    }
    threshold = newThr;
    @SuppressWarnings({"rawtypes","unchecked"})
    Node<K,V>[] newTab = (Node<K,V>[])new Node[newCap];
    table = newTab;
    if (oldTab != null) {
        for (int j = 0; j < oldCap; ++j) {
            Node<K,V> e;
            if ((e = oldTab[j]) != null) {
                oldTab[j] = null;
                if (e.next == null)
                    newTab[e.hash & (newCap - 1)] = e;
                else if (e instanceof TreeNode)
                    ((TreeNode<K,V>)e).split(this, newTab, j, oldCap);
                else { // preserve order
                    Node<K,V> loHead = null, loTail = null;
                    Node<K,V> hiHead = null, hiTail = null;
                    Node<K,V> next;
                    do {
                        next = e.next;
                        if ((e.hash & oldCap) == 0) {
                            if (loTail == null)
                                loHead = e;
                            else
                                loTail.next = e;
                            loTail = e;
                        }
                        else {
                            if (hiTail == null)
                                hiHead = e;
                            else
                                hiTail.next = e;
                            hiTail = e;
                        }
                    } while ((e = next) != null);
                    if (loTail != null) {
                        loTail.next = null;
                        newTab[j] = loHead;
                    }
                    if (hiTail != null) {
                        hiTail.next = null;
                        newTab[j + oldCap] = hiHead;
                    }
                }
            }
        }
    }
    return newTab;
}

public V get(Object key) {
    Node<K,V> e;
    // 我们进入getNode()看看
    return (e = getNode(hash(key), key)) == null ? null : e.value;
}

final Node<K,V> getNode(int hash, Object key) {
    Node<K,V>[] tab; Node<K,V> first, e; int n; K k;
    // 这里判断下table是否为null或者长度是否等于0，是的话直接返回null
    if ((tab = table) != null && (n = tab.length) > 0 &&
        (first = tab[(n - 1) & hash]) != null) {
        // 判断桶中(就是数组)元素是否相等，不相等就需要去桶中元素指定的链表/树去查找
        if (first.hash == hash && // always check first node
            ((k = first.key) == key || (key != null && key.equals(k))))
            return first;
        // 如果桶中有链表/树，就进入查找，没有就返回null
        if ((e = first.next) != null) {
            // 判断是否是树，是就进入树中查找
            if (first instanceof TreeNode)
                return ((TreeNode<K,V>)first).getTreeNode(hash, key);
            // 不然就是链表，循环遍历找到对应的元素
            do {
                if (e.hash == hash &&
                    ((k = e.key) == key || (key != null && key.equals(k))))
                    return e;
            } while ((e = e.next) != null);
        }
    }
    return null;
}

HashMap总结

  • HashMap的底层使用的是: 数组+链表+红黑树

  • HashMap中key和value都快可以为null

  • HashMap扩容的时机在于: 当前元素总个数(size) 大于 阈值(threshold)，进行扩容

    阈值(threshold) = 容量(capacity) * 装载因子(loadFactor)

    这个capacity就是table的容量，个人认为不就是table的长度吗？如果我理解的有偏差还请大家不吝赐教！!

  • HashMap转化成红黑树的时机: 当前某一个桶中的链表元素总个数大于 8，同时哈希表(table数组)长度大于 64，才会转化为红黑树，而不是扩容。

  • HashMap是线程不安全的，同时在jdk1.7中链表使用的是头插法，多线程操作下很容易出现死循环。jdk1.8使用的是尾插法，解决了这个问题，但是它仍然是线程不安全的。如果想保证线程请使用ConcurrentHashmap或者和HashTable(不建议HashTable)

4.ConcurrentHashmap简述

说实话，对我来说HashMap就已经很难了，所以ConcurrentHashmap我看起来依然十分吃力。源码好多地方我的水平还是很难弄懂，不过这个类的大致用法，以及部分重要的知识点我还是看的比较明白的，所以这里可能不会涉及太多的源码部分的讲解，还请大家多多包涵！

ConcurrentHashmap的存储结构~

  • 在JDK1.7中，它是介样滴~

图片来源: JavaGuide的PDF文档

地址: https://github.com/Snailclimb/JavaGuide

可以看到它的内部进一步细分了若干个小的HashMap，我们称作段(Segment)，默认情况下一个ConcurrentHashMap分为16段。这就表明了，如果不同线程操作的段不同是不会发生冲突的~

我是这么理解的，把Segment想象成一把锁，一个Segment数组就是一个锁数组，每个锁只锁住table数组中的一小段连续数据，这样就实现了多把锁控制同一个数组，访问数组中某一部分不会影响其他内容，加大了并发量~如有不对，还请指出呀！

Segment的结构和HashMap很相似，

  • 在JDK1.8中，它是介样滴~

在JDK1.8中取消使用Segment实现的分段锁，改为使用CAS+synchronized保证线程安全~由于只锁住一个节点，并发性进一步得到了提高！

ConcurrentHashMap的使用总结

  • 在JDK1.7中ConcurrentHashMap是使用Segment分段锁保证并发安全，而在JDK1.8中ConcurrentHashMap是使用CAS+synchronized保证并发安全。
  • ConcurrentHashMap中key和value都不允许存放null值
  • ConcurrentHashMap数据结构上也是哈希表(数组+链表)+红黑树，变为红黑树的时机也是当哈希表中的链表长度大于 8，哈希表的长度大于64时进行转换，否则先扩容
