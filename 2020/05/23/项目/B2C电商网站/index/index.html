1、 项目介绍

先来总结以下自己的开发路程吧，最初我是想做一个类似跳骚市场的在线网站给同学们用来进行一些二手用品的交易(因为学校有这个微信小程序，但是并没有这个网站，所以就想做一个这样的网站)。但是越开发，发现功能越多。。。导致到现在也没有完全开发完。又遇到了期末考，只能暂时搁置了，假期我会尽量把它补全。

ps：还是没经验啊，哎，前端大哥都被熬跑了。不求能真正投入使用，但还是希望能在毕业之前给学校做点贡献。

整个项目采用的是前后端分离式开发的：

  • 前端分为两部分：

    • 后台管理系统：主要面向的是后台管理员，采用基于Vue的单页应用开发方式
    • 前台系统：面向的是客户，前台采用的是Vue结合Nuxt实现服务端渲染方式

  • 后端—我负责

    后端采用基于SpringCloud的微服务架构，统一对外提供Rest风格接口，无论是后台管理系统还是前台系统都共享这些微服务接口，而微服务中通过JWT方式来识别用户身份，开放不同接口。

项目架构图如下所示，图是GitHub上扒的。因为是前后端分离式的，并且后端使用Spring Cloud微服务形式开发的，所以基本的核心微服务都是照着图上的微服务群中来做的。

3、 技术选型

后端技术：

我认为比较难的技术，会在后面标出用途

  • 基础的SpringMVC、Spring和MyBatis
  • Spring Boot
  • Spring Cloud：开发的核心，实现微服务开发。内置了Zuul网关、Eureka注册中心等重要部分。
  • Redis：实现可靠缓存服务即热点数据保存。
  • RabbitMQ：实现可靠消息服务，解决服务间通信问题。
  • Elasticsearch：高级聚合功能，实现商品的智能过滤搜索。
  • Nginx
  • FastDFS：解决大数据量的分布式文件存储问题。
  • Thymeleaf
  • JWT：JWT技术及RSA非对称加密实现无状态的单点登录。

前端技术：

  • 基础的HTML、CSS、JavaScript（基于ES6标准）
  • Vue.js 2.0以及基于Vue的UI框架：Vuetify
  • 前端构建工具：WebPack
  • 前端安装包工具：NPM
  • Vue脚手架：Vue-cli
  • Vue路由：vue-router
  • ajax框架：axios
  • 基于Vue的富文本框架：quill-editor

4、 微服务群

因为我是负责后端开发嘛，所以总结下我实现的微服务，用”微服务“表示已开发，”微服务”表示未开发。

  • 商品微服务：核心部分，对商品的增删改查都在此微服务内。
  • 订单微服务
  • 库存微服务
  • 搜索微服务：难点之一，主要用Elasticsearch实现智能搜索过滤。
  • 购物车微服务：难点之一，对购物车中的商品进行增删改查，并将用户添加到购物车中的商品存到redis中。
  • 用户认证微服务：JWT+RSA非对称加密以实现用户鉴权、服务鉴权。
  • 用户中心微服务：包含两部分，用户的登录和注册功能。
  • 支付微服务
  • 短信微服务：用于用户注册时，发送验证码
  • 评论微服务
  • 上传微服务：因为最初保存文件是保存到服务器，就会产生一些问题，比如单机器存储，存储能力有限、数据没有备份，有单点故障风险等等。了解到FastDFS后，采用FastDFS分布式文件系统存储文件。
